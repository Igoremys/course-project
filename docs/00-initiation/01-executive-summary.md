# ПАСПОРТ ПРОЕКТА
## 1. Общая информация
- **Название:** PhotoFolio – мобильное приложение для создания фотопортфолио
- **Траектория:** В (Мобильная разработка)
- **Автор:** Мусхажиев Игорь Александрович
- **Дата начала:** 05.05.2026
- **Плановая дата завершения:** 30.08.2026

## 2. Бизнес-контекст
- **Проблема:** Фотографам и креативным специалистам сложно быстро собирать, структурировать и безопасно делиться профессиональными портфолио без использования тяжёлых десктопных редакторов или облачных сервисов с платными подписками.
- **Решение:** Кроссплатформенное мобильное приложение с оффлайн-кэшированием, интуитивным редактором метаданных, автоматической генерацией ссылок для демонстрации и синхронизацией с облачным бэкендом.

## 3. Цели проекта
1. Реализовать PCMEF-архитектуру с чётким разделением слоёв.
2. Обеспечить работу в оффлайн-режиме с последующей синхронизацией.
3. Создать REST API с JWT-авторизацией и 8+ эндпоинтами.
4. Достичь покрытия модульными тестами >40%.

## 4. Ключевые показатели успеха
- Время загрузки портфолио ≤ 2 сек при 4G
- Покрытие кода тестами ≥ 45%
- Нулевые критические уязвимости в SonarQube
- 5+ экранов с Material Design 3

## 5. Ключевые риски
- **Риск:** Рассинхронизация локальной и серверной БД → **Реагирование:** Реализация Conflict Resolution по timestamp
- **Риск:** Превышение лимита хранилища на устройстве → **Реагирование:** Кэширование только превью, оригиналы по запросу
