## 🔷 SWOT-матрица

### Сильные стороны (Strengths) — внутренние преимущества
| № | Фактор | Обоснование |
|---|----------|-------------|
| S1 | **Архитектура PCMEF** | Чёткое разделение слоёв обеспечивает тестируемость, масштабируемость и лёгкость рефакторинга |
| S2 | **Оффлайн-режим через Room** | Локальное кэширование позволяет работать с портфолио без интернета — ключевое преимущество для фотографов на съёмках |
| S3 | **Нативная разработка на Kotlin** | Полный доступ к Android API, высокая производительность, лучшая поддержка камеры и работы с изображениями |
| S4 | **JWT-аутентификация** | Stateless-подход упрощает масштабирование бэкенда и обеспечивает безопасную авторизацию |
| S5 | **Material Design 3** | Современный, интуитивный интерфейс повышает UX и снижает порог входа для новых пользователей |

### Слабые стороны (Weaknesses) — внутренние недостатки
| № | Фактор | Обоснование |
|---|----------|-------------|
| W1 | **Сложность синхронизации** | Реализация Conflict Resolution при оффлайн-изменениях требует тщательного проектирования и тестирования |
| W2 | **Высокие требования к памяти** | Работа с полноформатными изображениями и превью может вызывать OOM на устройствах с малым объёмом RAM |
| W3 | **Двойная кодовая база** | Необходимость поддержки клиентского (Kotlin) и серверного (Java) кода увеличивает трудозатраты на разработку и отладку |
| W4 | **Зависимость от сторонних библиотек** | Retrofit, Room, Spring Security — обновления могут требовать адаптации кода, риск breaking changes |
| W5 | **Ограниченный функционал на старте** | Минимальная версия (MVP) не включает расширенные редакторы фото, что может снизить привлекательность для профессионалов |

### Возможности (Opportunities) — внешние благоприятные факторы
| № | Фактор | Потенциал |
|---|----------|-----------|
| O1 | **Рост рынка фриланс-услуг** | Увеличение числа фотографов, дизайнеров и креаторов, нуждающихся в быстром портфолио для демонстрации работ |
| O2 | **Интеграция с облачными хранилищами** | Возможность подключения Google Drive, Dropbox для резервного копирования и расширения хранилища |
| O3 | **Монетизация через PRO-функции** | Платные тарифы за расширенные возможности: экспорт в PDF, кастомные домены, аналитика просмотров |
| O4 | **Поддержка новых форматов** | Добавление работы с RAW, видео-превью, 360° фото — дифференциация от конкурентов |
| O5 | **Партнёрства с образовательными платформами** | Интеграция с курсами по фотографии для автоматического импорта учебных работ в портфолио |

### Угрозы (Threats) — внешние риски
| № | Фактор | Вероятность | Стратегия реагирования |
|---|----------|-------------|------------------------|
| T1 | **Конкуренция с готовыми решениями** (Behance, 500px, Adobe Portfolio) | Высокая | Фокус на нише «быстрое мобильное портфолио» + оффлайн-режим как УТП |
| T2 | **Изменения политик Google Play** (доступ к хранилищу, камере, фоновой работе) | Средняя | Следование Android Best Practices, регулярный аудит compliance |
| T3 | **Утечки персональных данных** | Низкая, но критическая | Шифрование чувствительных данных, регулярный аудит безопасности, минимизация собираемых данных |
| T4 | **Технические ограничения Android-фрагментации** | Средняя | Тестирование на минимальной поддерживаемой версии (API 24+), использование AndroidX |
| T5 | **Рост стоимости серверной инфраструктуры** | Средняя | Оптимизация хранения (сжатие превью, TTL для кэша), использование CDN для статики |

## 🔷 Стратегические выводы

### Стратегия SO (Сильные стороны + Возможности)
- Использовать нативную производительность (S3) для реализации поддержки RAW-форматов (O4)
- Применить PCMEF-архитектуру (S1) для быстрой интеграции с облачными хранилищами (O2)

### Стратегия WO (Слабые стороны + Возможности)
- Компенсировать сложность синхронизации (W1) за счёт внедрения отложенной фоновой синхронизации через WorkManager (O3)
- Снизить нагрузку на память (W2) через стриминговую загрузку превью и пагинацию (O1)

### Стратегия ST (Сильные стороны + Угрозы)
- Использовать оффлайн-режим (S2) как ключевое преимущество против облачных конкурентов (T1)
- Применить строгую валидацию и санитизацию входных данных (S4) для защиты от уязвимостей (T3)

### Стратегия WT (Слабые стороны + Угрозы)
- Минимизировать зависимость от сторонних библиотек (W4) через создание абстрактных репозиториев и моков для тестов (T4)
- Ограничить функционал MVP (W5) для снижения рисков несоответствия политикам магазинов приложений (T2)
