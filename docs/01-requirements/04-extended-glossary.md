## 1. Назначение документа
Данный документ содержит расширенный список терминов и определений, используемых в процессе проектирования и разработки системы PhotoFolio. Глоссарий объединяет бизнес-понятия предметной области, архитектурные концепции (PCMEF), а также технические определения стека (Kotlin, Android, Spring Boot), необходимые для однозначного понимания требований всеми участниками проекта.

---

## 2. Таблица терминов

| № | Термин | Определение |
|---|---|---|
| 1 | **REST API** | Архитектурный стиль взаимодействия приложений через протокол HTTP с использованием стандартных методов (GET, POST, PUT, DELETE) и передачей данных в формате JSON. |
| 2 | **JSON (JavaScript Object Notation)** | Текстовый формат обмена данными, основанный на JavaScript, используемый для сериализации и десериализации объектов при обмене информацией между клиентом и сервером. |
| 3 | **PCMEF** | Архитектурный паттерн (Presentation-Control-Mediator-Entity-Foundation), обеспечивающий строгое разделение ответственности и направленность зависимостей сверху вниз. |
| 4 | **Room Persistence Library** | Официальная библиотека персистентности Android, предоставляющая уровень абстракции над SQLite для безопасного кэширования данных и поддержки миграций. |
| 5 | **DAO (Data Access Object)** | Паттерн проектирования, инкапсулирующий доступ к базе данных. В проекте реализуется через интерфейсы Room для CRUD-операций с сущностями. |
| 6 | **Retrofit** | Типобезопасный HTTP-клиент для Android и Java, используемый для декларативного описания REST API-эндпоинтов и автоматической обработки JSON-ответов. |
| 7 | **StateFlow** | Класс из библиотеки Kotlin Coroutines, представляющий собой поток данных, удерживающий последнее состояние и рассылающий его коллекторам; используется для реактивного обновления UI. |
| 8 | **ViewModel** | Компонент архитектуры Android Jetpack, предназначенный для хранения и управления UI-данными с учетом жизненного цикла экранов (сохранение при повороте). |
| 9 | **Jetpack Compose** | Современный инструментарий Android для построения нативных UI декларативным способом, заменяющий традиционные XML-макеты. |
| 10 | **Kotlin Coroutines** | Концепция сопрограмм в Kotlin для написания асинхронного кода в последовательном стиле, используемая для фоновых задач и сетевых запросов. |
| 11 | **Dependency Injection (DI)** | Паттерн внедрения зависимостей, при котором объекты получают свои зависимости извне, а не создают их самостоятельно; в Android реализуется через Hilt/Dagger. |
| 12 | **JWT (JSON Web Token)** | Стандарт (RFC 7519) для создания токенов доступа, позволяющий передавать информацию между сторонами в компактном и самодостаточном виде, защищенном подписью. |
| 13 | **BCrypt** | Адаптивная криптографическая хеш-функция на основе шифра Blowfish, используемая для безопасного хранения паролей пользователей в базе данных. |
| 14 | **Spring Boot** | Фреймворк для создания автономных Java-приложений, используемый на сервере для быстрой настройки Spring-контекста, REST-контроллеров и безопасности. |
| 15 | **JPA (Java Persistence API)** | Спецификация Java для ORM, позволяющая управлять реляционными данными в приложении с помощью объектов Java, скрывая детали SQL-взаимодействия. |
| 16 | **OpenAPI Specification** | Стандарт описания REST API в формате JSON или YAML, позволяющий генерировать интерактивную документацию (Swagger UI) и клиенты. |
| 17 | **CRUD** | Аббревиатура (Create, Read, Update, Delete), описывающая четыре базовые операции управления данными в постоянном хранилище. |
| 18 | **Нормализация (3NF)** | Процесс организации данных в БД для уменьшения дублирования и обеспечения целостности, при котором неключевые атрибуты зависят только от первичного ключа. |
| 19 | **Conflict Resolution** | Стратегия разрешения конфликтов данных, возникающих при оффлайн-синхронизации, когда запись была изменена одновременно на клиенте и сервере. |
| 20 | **Offline-First** | Архитектурный подход, при котором приложение проектируется для работы в первую очередь с локальной копией данных, синхронизируясь с сервером при возможности. |
| 21 | **Token Refresh** | Механизм обновления истекшего JWT-токена без повторного ввода логина и пароля, обычно использующий специальный Refresh Token с длительным сроком жизни. |
| 22 | **Lazy Loading** | Паттерн отложенной загрузки связанных объектов (например, списка фото внутри альбома) для оптимизации потребления памяти и ускорения отклика. |
| 23 | **Middleware** | Промежуточный слой в обработке HTTP-запросов на сервере (например, фильтры безопасности), выполняющий общие задачи (логирование, авторизация) до вызова контроллера. |
| 24 | **Material Design 3** | Система визуального дизайна и рекомендаций от Google, обеспечивающая единообразие и удобство пользовательского опыта в Android-приложениях. |
